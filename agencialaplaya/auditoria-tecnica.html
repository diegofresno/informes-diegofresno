Crítico T4

.htaccess — redirect HTTP → HTTPS (301 permanente)

✓ Hecho · 2026-07-20
Estado actual

La versión HTTP del sitio está indexada y posiciona mejor que la HTTPS (HTTP pos. 7,02 · HTTPS pos. 14,73). La autoridad está partida entre dos versiones del mismo dominio. El redirect 301 de HTTP → HTTPS no se está aplicando correctamente a nivel servidor. El .htaccess del sitio ya tiene RewriteEngine on activo, lo que nos permite corregirlo directamente via FTP.

Edición del .htaccess — añadir 2 líneas al inicio

RewriteEngine on
# Forzar HTTPS — redirige cualquier petición HTTP a HTTPS
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [R=301,L]

# Resto de reglas existentes sin modificar
RewriteCond %{REQUEST_URI} !^/administra [NC]
# ...
Acción

Descargar el .htaccess actual via FTP, añadir las dos líneas (RewriteCond + RewriteRule) justo después de RewriteEngine on y subir. Verificar con curl -I http://www.agencialaplaya.com/ que devuelve 301 → https://. Efecto: Google consolida toda la autoridad acumulada en la versión HTTPS en pocos días de rastreo.

Bing Webmaster Tools — verificación del sitio

Por qué importa

Bing es el motor que alimenta el módulo de búsqueda de ChatGPT y Microsoft Copilot. Sin estar en Bing Webmaster Tools, estas plataformas tienen mucha menos visibilidad del sitio.

Acción

César accede a bing.com/webmasters y genera el archivo XML de verificación (BingSiteAuth.xml). Lo subimos via FTP a la raíz y completamos la verificación desde su panel.

S1 Soft 404 → 410/404 en fichas de inmueble inexistentes o vendidas Crítico Actualmente cualquier URL /inmueble/{slug}-{id} que no exista redirige con 301 a la home. Lo correcto: ID activo → 200, ID vendido/reservado → 410 Gone con página "vivienda no disponible", ID inexistente → 404 limpio.
